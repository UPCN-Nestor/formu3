Tenemos un sistema de RRHH arcaico, que para la liquidación de sueldos tiene fórmulas con "variables" (en realidad funciones) horribles, que referencian conceptos calculados anteriormente (se hace una sola pasada).

Ej. del diccionario de variables: 
* Patrón: CALCnnnn - Descripción: IMPORTE CALCULADO EN EL CONCEPTO INDICADO EN nnnn - Texto para mostrar: "Valor de nnnn"

Quiero desarrollar una GUI web que muestre cada concepto en un rectángulo de color (basado en el hash del número de concepto, para que cada concepto siempre tenga el mismo color), con su respectiva fórmula. Los conceptos definitivos van a tener borde doble. Una fórmula puede tener varias variables, ej:
(%CALC3498% / 180,00) * %ST00340035%
En este caso, cada variable va resaltada de un color distinto (basado en el número de concepto que referencia, o en la variable en los casos en que involucre más de un concepto como "SC"). De ahí sale una flecha hacia arriba hacia un nuevo rectángulo pintado del color correcto, con la información del concepto corerespondiente. 
Entonces cada concepto tendría un árbol de conceptos de los cuales, y también un árbol "inverso", hacia abajo, de conceptos que dependen de él. Para esto último va a haber que armar algún tipo de caché, porque va a ser una búsqueda pesada.
Cada variable mostraría su "texto para mostrar", de manera tal que "%CALC7680%" se convertiría en "Valor de 7680"; si no está disponible, se muestra la el nombre de la variable, ej. "CALC de nnnn".

Algunas variables involucran más de un concepto, por ej. "SC", sumatoria de conceptos. En este caso, el color de la variable va a ser según el hash de la variable completa (ej. "SC01003600") y va a apuntar a un rectángulo especial de ese mismo color, que va a tener una lista de los conceptos referenciados, cada uno con un "+" al lado, que va a permitir ampliar y ver ese concepto de la misma forma que hablamos anteriormente.

Otras variables referencian cuestiones que exceden esta implementación y van a ser "terminales".

Cada vez que se hace click en un rectángulo, se cargan sus dependencias y dependientes (si no están ya). Hasta cierta cantidad de rectángulos, configurable, supongo.
En este "plano" se tiene que permitir hacer zoom hacia adentro y hacia afuera, y a scrollear.

Se tienen que poder guardar pantallas, con un nombre sugerido (según el concepto raíz supongo), y volver a abrirlas a partir de un desplegable.

También necesito la opción de cargar Año, Mes (default actuales), Tipo Liquidación, Legajo (default todos) y a partir de eso buscar en una tabla de liquidación los importes reales que se liquidaron, y adorsarlo a cada rectángulo de forma destacada.

El sistema tiene que responder rápidamente, tanto en el aspecto gráfico como en cuanto a la carga de datos. Sugerir las tecnologías apropiadas para este requerimiento.

CREATE VIEW [dbo].[ConceptoTipoLiqFormula] AS
SELECT DISTINCT C.ConCod CodConcepto, C.ConDsc DescripcionConcepto, 
	C.ConTra TransitorioDefinitivo, 
	TL.TpoLiqDsc TipoLiquidacion, 
	C.ConOrd Orden, 
	CASE 
		WHEN C.ConTpoCnc = 1 THEN '1: Sujeto a Retención'
		WHEN C.ConTpoCnc = 2 THEN '2: No Sujeto a Retención'
		WHEN C.ConTpoCnc = 3 THEN '3: Salario Familiar'
		WHEN C.ConTpoCnc = 4 THEN '4: Retenciones'
		WHEN C.ConTpoCnc = 5 THEN '5: Deducciones Varias'
		WHEN C.ConTpoCnc = 6 THEN '6: Aportes'
		WHEN C.ConTpoCnc = 7 THEN '7: Provisión'
		ELSE 'Unknown' -- or any other default value you prefer
	END AS TipoConcepto,
	F.ForCod CodFormula, F.ForDsc DescripcionFormula,
	ForFor1 + ForFor2 + ForFor3 FormulaCompleta,
	ForPara11 + ForCompa1 + ' ' + ForPara12 + ForUnion1 + ' ' + ForPara21 + ForCompa2 + ' ' + ForPara22 CondicionFormula
FROM [UPCN_RRHH_PRUE].dbo.[FORMUL] F
	JOIN [UPCN_RRHH_PRUE].dbo.[CONCE21] C
		ON (F.SucCod = C.SucCod and C.ForCod = F.ForCod)
	JOIN [UPCN_RRHH_PRUE].dbo.[TIPLIQ] TL
		ON (C.ConTpoLiq = TL.TpoLiqCod)
WHERE F.ForVig = '01/01/1753' and C.ConVig = '01/01/1753' 
	AND C.SucCod = 1
